🛠️ Cómo Reconstruir este Proyecto Desde Cero

Esta guía te lleva paso a paso para crear tu propia versión de la Copa Mundial FIFA 2026. Sigue cada paso en orden.

Paso 1: Crear la estructura de carpetas

Crea las carpetas del proyecto:

WC2026_Prueba/
├── js/
│   └── i18n/
│       └── es/
├── matches/
├── groups/
├── knockout/
├── stats/
└── learn/

Cada carpeta tendrá un archivo HTML y un archivo JS (excepto js/ que tiene los scripts compartidos).

Crea tailwindCustom.js con la configuración de colores, fuentes y espaciado (copia la paleta de Material Design 3 púrpura del proyecto original).

Paso 4: Configurar colores y fuentes en tailwindCustom.js

Define los colores usando la paleta Material Design 3. Los principales son:

  • primary: #4f378a (púrpura oscuro) — Color principal de la marca
  • background: #fdf7ff (blanco con tono púrpura) — Fondo general
  • surface: #fdf7ff — Superficie de tarjetas y contenedores
  • inverse-surface: #322f35 (gris oscuro) — Footer y fondos invertidos

Fuentes: Montserrat para títulos (display, headline), Inter para cuerpo (body, label).

Paso 5: Crear el menú móvil (menu.js)

El menú móvil se comparte entre todas las páginas. Crea js/menu.js con una IIFE que:

  1. Busca los botones open-menu y close-menu por ID
  2. Muestra/oculta el overlay del menú
  3. Bloquea el scroll del body cuando el menú está abierto
  4. Cierra con tecla Escape o clic fuera del menú

Paso 6: Crear el proxy PHP (proxy.php)

Para evitar el problema CORS, crea un archivo PHP que actúe como intermediario:

Necesitas un token gratuito de football-data.org.

Paso 7: Conectar con la API (api.js)

Crea js/api.js con funciones para cada endpoint:

  • fetchApi(endpoint) — Función universal con caché, proxy y fallback directo
  • getAllMatches() — Todos los partidos
  • getMatchesByStatus(status) — Partidos filtrados por estado
  • getStandings() — Clasificaciones
  • getGroupMatches() — Partidos de fase de grupos
  • getMatchesByStage(stage) — Partidos por etapa (eliminatorias)
  • getScorers() — Goleadores
  • getTeams() — Equipos

Paso 8: Implementar el sistema de caché

Dentro de api.js, agrega funciones para guardar en localStorage:

  • getFromCache(key) — Busca en localStorage, verifica si expiró
  • setToCache(key, data) — Guarda con fecha de expiración (5 min)
  • cleanupCache() — Limpia entradas antiguas

Paso 9: Agregar traducciones al español (i18n)

Crea los archivos de diccionario en js/i18n/es/:

  • teams.js — Mapa de códigos FIFA a nombres en español
  • enums.js — Traducciones de estados (SCHEDULED → Programado), etapas, etc.
  • competition.js — Términos del torneo
  • ui.js — Textos de la interfaz (HOME, LOADING, ERROR...)
  • index.js — Combina todos los diccionarios en window.__DICT

Luego crea js/i18n/i18n.js con las funciones de traducción: t(), translateTeam(), translateStatus(), formatDate(), etc.

Paso 10: Crear cada página

Para cada página (matches, groups, knockout, stats):

  1. Crea el HTML con la estructura (header, main, footer, scripts)
  2. Crea el JS con las funciones de renderizado
  3. Incluye los scripts compartidos en este orden: teams.js → enums.js → competition.js → ui.js → index.js → i18n.js → menu.js → api.js → [tu-pagina].js
  4. Usa addEventListener('DOMContentLoaded', tuFuncion) para que el código se ejecute al cargar la página

Paso 11: Probar localmente

Abre los archivos HTML en tu navegador. Para que el proxy PHP funcione, necesitas un servidor PHP local:

# Si tienes PHP instalado:
php -S localhost:8000

# Luego abre http://localhost:8000 en tu navegador

Sin PHP, las páginas cargarán pero los datos de la API se obtendrán mediante fetch directo (si el CORS lo permite desde tu dominio).

Paso 12: Desplegar en GitHub Pages

Para publicar tu proyecto:

  1. Sube el código a un repositorio de GitHub
  2. Ve a Settings → Pages
  3. Selecciona la rama main y la carpeta / (root)
  4. ¡Tu sitio estará disponible en https://tusuario.github.io/WC2026_Prueba/!

Nota: En GitHub Pages el proxy PHP no funcionará (no soporta PHP). Los datos se obtendrán mediante fetch directo. Si la API lo permite, funcionará. Si no, puedes usar un servicio serverless o un proxy como corsproxy.io.

🎉 ¡Felicidades! Has reconstruido el proyecto completo. Ahora entiendes cómo funciona cada parte. Puedes modificarlo, mejorarlo y crear tus propias versiones.

🛠️ How to Rebuild This Project From Scratch

This guide takes you step by step to create your own version of the FIFA World Cup 2026 site. Follow each step in order.

Step 1: Create the folder structure

Create the project folders.

Step 2: Set up Tailwind CSS

Load Tailwind from CDN in every page's . Create tailwindCustom.js with colors, fonts, and spacing config (purple Material Design 3 palette).

Step 3: Create the base HTML (index.html)

Basic page structure with html, head, body, header, main, footer.

Step 4: Configure colors and fonts in tailwindCustom.js

Define the Material Design 3 palette: primary #4f378a, background #fdf7ff, inverse-surface #322f35. Fonts: Montserrat for headings, Inter for body.

Step 5: Create the mobile menu (menu.js)

Create an IIFE that handles open/close, body scroll lock, Escape key, and click-outside-to-close.

Step 6: Create the PHP proxy (proxy.php)

To avoid CORS issues, create a PHP intermediary file. Get a free API token from football-data.org.

Step 7: Connect to the API (api.js)

Create functions for each endpoint: getAllMatches, getStandings, getScorers, etc. Include cache logic.

Step 8: Implement the cache system

Add localStorage functions with 5-minute TTL, automatic cleanup, and JSON serialization.

Step 9: Add Spanish translations (i18n)

Create dictionary files for teams, enums, competition, and UI. Build the translation engine with t(), translateTeam(), formatDate(), etc.

Step 10: Create each page

For each page (matches, groups, knockout, stats): create HTML structure and JS rendering functions. Include shared scripts in correct order.

Step 11: Test locally

Open HTML files in browser. For PHP proxy, use: php -S localhost:8000. Without PHP, pages still load but API calls use direct fetch.

Step 12: Deploy to GitHub Pages

Push code to GitHub, enable Pages (main branch, root folder). Note: PHP proxy won't work on GitHub Pages — data will use direct fetch or a CORS proxy service.
