🔌 API & Proxy PHP

¿Qué es una API?

API significa Application Programming Interface (Interfaz de Programación de Aplicaciones). Es como un camarero en un restaurante: tú le pides algo (datos) y él va a la cocina (el servidor) y te trae lo que pediste.

En nuestro proyecto, la API es https://api.football-data.org/v4. Le pedimos cosas como:

  • /competitions/WC — Información general del Mundial
  • /competitions/WC/matches — Todos los partidos
  • /competitions/WC/standings — Clasificaciones
  • /competitions/WC/scorers — Máximos goleadores
  • /competitions/WC/teams — Equipos participantes

La API devuelve los datos en formato JSON (JavaScript Object Notation), que es como un objeto de JavaScript.

El Problema CORS

CORS (Cross-Origin Resource Sharing) es una medida de seguridad del navegador. Impide que una página web (por ejemplo, https://tusitio.com) haga peticiones a un dominio diferente (como api.football-data.org).

Imagina que estás en tu casa (tu página web) y quieres pedir pizza de una pizzería diferente (la API). El navegador dice: "No, no puedes pedir directamente. Tienes que ir tú mismo o usar un servicio de mensajería."

Ese servicio de mensajería es nuestro proxy.php.

proxy.php — El Mensajero

El archivo proxy.php está en el mismo servidor que nuestra página web. Cuando JavaScript necesita datos de la API:

  1. JS llama a proxy.php?endpoint=/competitions/WC/matches
  2. proxy.php recibe la petición en el servidor
  3. proxy.php hace una llamada interna (server-to-server) a la API real — esto NO tiene restricciones CORS
  4. proxy.php devuelve los datos a JavaScript

Código simplificado del proxy:

// Recibir la petición de JavaScript
$endpoint = $_GET['endpoint'];
$url = 'https://api.football-data.org/v4' . $endpoint;

// Hacer la llamada a la API (con curl o file_get_contents)
$response = curl_exec($ch);

// Devolver los datos al navegador
header('Content-Type: application/json');
echo $response;

Dos Métodos en PHP: Curl vs file_get_contents

PHP tiene dos formas de hacer peticiones a otras URLs:

  • cURL — La mejor opción. Más rápido, más configurable, soporta más protocolos. Se usa con curl_init(), curl_setopt(), curl_exec().
  • file_get_contents — Más simple, pero requiere configurar un "contexto" especial. Es el plan de respaldo si curl no está instalado.

El proxy intenta primero con curl. Si no está disponible, usa file_get_contents.

Estrategia en JavaScript: Proxy + Fallback

En api.js, la función fetchApi() sigue esta estrategia:

  1. Verificar caché (localStorage)
  2. Si no hay caché, intentar con proxy.php
  3. Si el proxy falla, intentar directamente con la API
  4. Guardar el resultado en caché

🔌 API & PHP Proxy

What is an API?

API stands for Application Programming Interface. It's like a waiter at a restaurant: you ask for something (data), and they go to the kitchen (the server) and bring you what you ordered.

In our project, the API is https://api.football-data.org/v4. We ask it for things like:

  • /competitions/WC — General World Cup info
  • /competitions/WC/matches — All matches
  • /competitions/WC/standings — Standings
  • /competitions/WC/scorers — Top scorers
  • /competitions/WC/teams — Participating teams

The API returns data in JSON format (JavaScript Object Notation), which is like a JavaScript object.

The CORS Problem

CORS (Cross-Origin Resource Sharing) is a security measure in browsers. It prevents a web page (e.g. https://yoursite.com) from making requests to a different domain (like api.football-data.org).

Imagine you're at home (your website) and want to order pizza from a different pizzeria (the API). The browser says: "No, you can't order directly. You have to go yourself or use a delivery service."

That delivery service is our proxy.php.

proxy.php — The Messenger

The proxy.php file is on the same server as our website. When JavaScript needs data from the API:

  1. JS calls proxy.php?endpoint=/competitions/WC/matches
  2. proxy.php receives the request on the server
  3. proxy.php makes an internal (server-to-server) call to the real API — this has NO CORS restrictions
  4. proxy.php returns the data to JavaScript

Simplified proxy code:

// Receive the request from JavaScript
$endpoint = $_GET['endpoint'];
$url = 'https://api.football-data.org/v4' . $endpoint;

// Make the API call (with curl or file_get_contents)
$response = curl_exec($ch);

// Return data to the browser
header('Content-Type: application/json');
echo $response;

Two PHP Methods: Curl vs file_get_contents

PHP has two ways to make requests to other URLs:

  • cURL — The best option. Faster, more configurable, supports more protocols. Uses curl_init(), curl_setopt(), curl_exec().
  • file_get_contents — Simpler, but requires setting up a special "context". It's the backup plan if curl is not installed.

The proxy tries curl first. If not available, it uses file_get_contents.

JavaScript Strategy: Proxy + Fallback

In api.js, the fetchApi() function follows this strategy:

  1. Check cache (localStorage)
  2. If no cache, try proxy.php
  3. If proxy fails, try direct API call
  4. Save result in cache
