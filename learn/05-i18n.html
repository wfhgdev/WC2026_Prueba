🌐 Internacionalización (i18n)

¿Qué es i18n?

i18n es la abreviatura de internacionalización (hay 18 letras entre la i y la n). Es un sistema que permite que una aplicación web se muestre en diferentes idiomas.

En nuestro proyecto, todo el texto de la interfaz está en español. Los nombres de los equipos ("Brazil" → "Brasil"), los estados de los partidos ("FINISHED" → "Finalizado"), las fechas, etc.

¿Cómo funciona?

El sistema i18n funciona con diccionarios. Un diccionario es un objeto JavaScript que asocia una clave (en inglés) con un valor (en español):

window.__TEAMS_ES = {
  BRA: 'Brasil',
  ARG: 'Argentina',
  ESP: 'España',
  // ... más de 200 equipos
};

Luego, una función t(clave) busca la clave en el diccionario y devuelve el valor en español:

function t(key) {
  return dict[key] || key; // Si no encuentra, devuelve la clave
}

t('BRA'); // "Brasil"
t('HELLO'); // "HELLO" (no está en el diccionario)

Los Archivos de Traducción

Hay 4 archivos de diccionario en js/i18n/es/:

  • teams.js — Nombres de equipos por código FIFA (BRA, ARG, ESP...)
  • enums.js — Estados (SCHEDULED, IN_PLAY, FINISHED), etapas (FINAL, SEMI_FINALS), prefijo de grupo
  • competition.js — Términos de la competición (WORLD_CUP, SEASON, TOURNAMENT)
  • ui.js — Textos de la interfaz (HOME, MATCHES, LOADING, ERROR...)

Luego index.js combina los 4 diccionarios en uno solo: window.__DICT

El Archivo i18n.js — El Motor

El archivo js/i18n/i18n.js contiene el "motor" de traducción. Expone un objeto global i18n con estas funciones principales:

  • t(clave) — Traduce una clave genérica
  • translateTeam(equipo) — Traduce el nombre de un equipo (por TLA o nombre)
  • translateStatus(estado) — Traduce el estado de un partido
  • translateStage(etapa) — Traduce el nombre de una etapa (Semi_finals → Semifinales)
  • translateGroup(grupo) — Traduce un grupo ("Grupo A", "Grupo 1")
  • formatDate(fecha) — Formatea una fecha en español ("lunes, 15 de junio de 2026")
  • formatTime(fecha) — Formatea la hora ("21:00")
  • getRelativeDay(fecha) — Dice "Hoy", "Mañana", "Ayer" o la fecha

El Mapa NAME_TO_TLA

A veces la API devuelve el nombre del equipo en inglés ("Brazil", "Argentina") en lugar del código de 3 letras (BRA, ARG).

El NAME_TO_TLA es un mapa que convierte nombres en inglés a códigos FIFA:

var NAME_TO_TLA = {
  'Brazil': 'BRA',
  'Argentina': 'ARG',
  'Spain': 'ESP',
  'United States': 'USA',
  'Côte d\'Ivoire': 'CIV'
};

Así podemos buscar cualquier equipo por su nombre y obtener su traducción al español.

🌐 Internationalization (i18n)

What is i18n?

i18n is short for internationalization (there are 18 letters between i and n). It's a system that allows a web application to display in different languages.

In our project, all interface text is in Spanish. Team names ("Brazil" → "Brasil"), match statuses ("FINISHED" → "Finalizado"), dates, etc.

How does it work?

The i18n system uses dictionaries. A dictionary is a JavaScript object that maps a key (in English) to a value (in Spanish):

window.__TEAMS_ES = {
  BRA: 'Brasil',
  ARG: 'Argentina',
  ESP: 'España',
  // ... more than 200 teams
};

Then a t(key) function looks up the key in the dictionary and returns the Spanish value:

function t(key) {
  return dict[key] || key; // If not found, return the key
}

t('BRA'); // "Brasil"
t('HELLO'); // "HELLO" (not in dictionary)

Translation Files

There are 4 dictionary files in js/i18n/es/:

  • teams.js — Team names by FIFA code (BRA, ARG, ESP...)
  • enums.js — Statuses (SCHEDULED, IN_PLAY, FINISHED), stages (FINAL, SEMI_FINALS), group prefix
  • competition.js — Competition terms (WORLD_CUP, SEASON, TOURNAMENT)
  • ui.js — Interface texts (HOME, MATCHES, LOADING, ERROR...)

Then index.js combines the 4 dictionaries into one: window.__DICT

The i18n.js File — The Engine

The js/i18n/i18n.js file contains the translation "engine". It exposes a global i18n object with these main functions:

  • t(key) — Translates a generic key
  • translateTeam(team) — Translates a team name (by TLA or name)
  • translateStatus(status) — Translates a match status
  • translateStage(stage) — Translates a stage name
  • translateGroup(group) — Translates a group ("Grupo A", "Grupo 1")
  • formatDate(date) — Formats a date in Spanish
  • formatTime(date) — Formats time ("21:00")
  • getRelativeDay(date) — Says "Hoy", "Mañana", "Ayer" or the date

The NAME_TO_TLA Map

Sometimes the API returns the team name in English ("Brazil", "Argentina") instead of the 3-letter code (BRA, ARG).

The NAME_TO_TLA is a map that converts English names to FIFA codes:

var NAME_TO_TLA = {
  'Brazil': 'BRA',
  'Argentina': 'ARG',
  'Spain': 'ESP',
  'United States': 'USA',
  'Côte d\'Ivoire': 'CIV'
};

This way we can look up any team by its name and get its Spanish translation.
