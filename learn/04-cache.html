💾 Sistema de Caché

¿Qué es la Caché?

La caché es un almacenamiento temporal de datos. En lugar de pedir los mismos datos a la API cada vez que entras a una página, los guardamos en el navegador y los reutilizamos durante 5 minutos.

¿Por qué es importante? Porque:

  • La página carga más rápido (no espera la respuesta de la API)
  • Ahorramos ancho de banda (menos peticiones a internet)
  • La API no nos limita por hacer demasiadas peticiones

localStorage — El Almacén del Navegador

El navegador tiene un pequeño almacén llamado localStorage. Es como una mochila donde puedes guardar cosas (datos) y recuperarlas después, incluso si cierras la página y vuelves a abrirla.

// Guardar algo
localStorage.setItem('clave', 'valor');

// Recuperar algo
const dato = localStorage.getItem('clave');

// Eliminar algo
localStorage.removeItem('clave');

// Limpiar todo
localStorage.clear();

Pero localStorage solo guarda texto. Para guardar objetos complejos, usamos JSON.stringify() (convertir a texto) y JSON.parse() (convertir de texto a objeto).

La Estrategia de Caché en 4 Pasos

En api.js, la función fetchApi() implementa esta estrategia:

  1. Verificar caché — Buscar en localStorage si ya tenemos estos datos guardados y no han expirado
  2. Si hay caché válida — Devolver los datos inmediatamente (sin llamar a la API)
  3. Si no hay caché — Llamar a la API (a través del proxy o directamente)
  4. Guardar en caché — Almacenar los datos con una fecha de expiración (5 minutos)

TTL (Time To Live)

Cada entrada en la caché tiene un TTL de 5 minutos. Esto significa que después de 5 minutos, los datos se consideran "caducados" y se vuelven a pedir a la API.

const CACHE_CONFIG = {
  ttl: 5 * 60 * 1000,  // 5 minutos en milisegundos
  prefix: 'wc2026_cache_',
  maxEntries: 30
};

Cuando guardamos datos, también guardamos expiresAt (cuándo expira). Al recuperar, comprobamos si la fecha actual es posterior a expiresAt.

Limpieza Automática

Para que la caché no crezca infinitamente, hay una función cleanupCache() que:

  • Elimina entradas expiradas
  • Si hay más de 30 entradas, elimina las más antiguas
  • Se ejecuta automáticamente antes de guardar una nueva entrada

💾 Cache System

What is Cache?

Cache is a temporary storage for data. Instead of requesting the same data from the API every time you enter a page, we save it in the browser and reuse it for 5 minutes.

Why is it important? Because:

  • The page loads faster (no waiting for API response)
  • We save bandwidth (fewer internet requests)
  • The API doesn't limit us for making too many requests

localStorage — The Browser's Storage

The browser has a small storage called localStorage. It's like a backpack where you can store things (data) and retrieve them later, even if you close the page and reopen it.

// Save something
localStorage.setItem('key', 'value');

// Retrieve something
const data = localStorage.getItem('key');

// Delete something
localStorage.removeItem('key');

// Clear everything
localStorage.clear();

But localStorage only stores text. To save complex objects, we use JSON.stringify() (convert to text) and JSON.parse() (convert from text to object).

The 4-Step Cache Strategy

In api.js, the fetchApi() function implements this strategy:

  1. Check cache — Look in localStorage if we already have this data saved and it hasn't expired
  2. If valid cache exists — Return the data immediately (without calling the API)
  3. If no cache — Call the API (through the proxy or directly)
  4. Save to cache — Store the data with an expiration date (5 minutes)

TTL (Time To Live)

Each cache entry has a TTL of 5 minutes. This means after 5 minutes, the data is considered "expired" and is requested from the API again.

const CACHE_CONFIG = {
  ttl: 5 * 60 * 1000,  // 5 minutes in milliseconds
  prefix: 'wc2026_cache_',
  maxEntries: 30
};

When we save data, we also save expiresAt (when it expires). When retrieving, we check if the current date is past expiresAt.

Automatic Cleanup

To prevent the cache from growing infinitely, there's a cleanupCache() function that:

  • Removes expired entries
  • If there are more than 30 entries, removes the oldest ones
  • Runs automatically before saving a new entry
