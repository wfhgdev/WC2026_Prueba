⚡ JavaScript

¿Qué es JavaScript?

JavaScript (JS) es el lenguaje de programación que hace que las páginas web sean interactivas. Sin JS, una web sería solo texto e imágenes estáticas. Con JS, podemos:

  • Responder a clics de botones
  • Obtener datos de internet (APIs)
  • Crear y modificar elementos HTML en vivo
  • Guardar información en el navegador
  • Animar elementos

En este proyecto usamos JavaScript Vanilla — es decir, JavaScript puro, sin frameworks como React, Vue o Angular.

El DOM (Document Object Model)

El DOM es una representación del HTML en forma de árbol que JavaScript puede modificar. Piensa en él como un árbol genealógico: la etiqueta es el abuelo, es el padre, y cada

es un hijo.

Métodos principales para trabajar con el DOM:

// Buscar un elemento por su ID
const menu = document.getElementById('mobile-menu');

// Buscar varios elementos por su clase
const cards = document.querySelectorAll('.match-card');

// Crear un nuevo elemento
const div = document.createElement('div');

// Añadir una clase
div.classList.add('hidden');

// Escuchar un evento (clic)
button.addEventListener('click', function() {
  alert('¡Hiciste clic!');
});

Funciones

Una función es un bloque de código reutilizable. En lugar de escribir el mismo código varias veces, escribes una función y la llamas cuando la necesitas.

// Declaración de función tradicional
function saludar(nombre) {
  return 'Hola, ' + nombre + '!';
}

// Llamar a la función
const mensaje = saludar('Ana'); // "Hola, Ana!"

// Función flecha (arrow function) - más moderna
const sumar = (a, b) => a + b;

En el proyecto, usamos funciones para todo: renderMatches(), fetchApi(), calculateTeamStats(), etc.

Async / Await — Código que Espera

Cuando pedimos datos a una API, el código no puede esperar parado — la página se congelaría. Async/await es una forma de escribir código que "espera" sin congelar la página.

// async = esta función es asíncrona (tarda tiempo)
async function obtenerDatos() {
  try {
    // await = espera a que la promesa se resuelva
    const respuesta = await fetch('https://api.ejemplo.com/datos');
    const datos = await respuesta.json();
    return datos;
  } catch (error) {
    // Si algo sale mal, capturamos el error
    console.error('Error:', error);
  }
}

En nuestro proyecto, fetchApi() en api.js usa async/await para llamar a la API de fútbol.

Fetch API — Llamadas a Internet

fetch() es una función de JavaScript para hacer peticiones HTTP (pedir datos a un servidor). Es como escribir una URL en el navegador, pero desde el código.

// Petición GET simple
const response = await fetch('https://api.football-data.org/v4/competitions/WC/matches');
const data = await response.json(); // Convertir respuesta a JSON

El proyecto usa fetch de dos maneras:

  1. A través del proxy PHP (método principal) — fetch('proxy.php?endpoint=...')
  2. Directa (fallback) — fetch('https://api.football-data.org/...', { headers: { 'X-Auth-Token': '...' } })

Promise.all — Varias Llamadas a la Vez

Cuando necesitamos hacer varias peticiones a la API que no dependen una de la otra, podemos hacerlas en paralelo para ahorrar tiempo:

const [datos1, datos2, datos3] = await Promise.all([
  fetch('/api/recurso1'),
  fetch('/api/recurso2'),
  fetch('/api/recurso3')
]);

En knockout.js usamos Promise.all para obtener todas las rondas del torneo simultáneamente.

⚡ JavaScript

What is JavaScript?

JavaScript (JS) is the programming language that makes web pages interactive. Without JS, a website would just be static text and images. With JS, we can:

  • Respond to button clicks
  • Get data from the internet (APIs)
  • Create and modify HTML elements live
  • Save information in the browser
  • Animate elements

In this project we use Vanilla JavaScript — pure JavaScript, no frameworks like React, Vue, or Angular.

The DOM (Document Object Model)

The DOM is a tree representation of HTML that JavaScript can modify. Think of it like a family tree: is the grandparent, is the parent, and each

is a child.

Main methods for working with the DOM:

// Find an element by its ID
const menu = document.getElementById('mobile-menu');

// Find multiple elements by class
const cards = document.querySelectorAll('.match-card');

// Create a new element
const div = document.createElement('div');

// Add a class
div.classList.add('hidden');

// Listen for an event (click)
button.addEventListener('click', function() {
  alert('You clicked!');
});

Functions

A function is a reusable block of code. Instead of writing the same code multiple times, you write a function and call it when needed.

// Traditional function declaration
function greet(name) {
  return 'Hello, ' + name + '!';
}

// Call the function
const message = greet('John'); // "Hello, John!"

// Arrow function - more modern
const add = (a, b) => a + b;

In the project, we use functions for everything: renderMatches(), fetchApi(), calculateTeamStats(), etc.

Async / Await — Code that Waits

When we request data from an API, the code can't just stop and wait — the page would freeze. Async/await is a way to write code that "waits" without freezing the page.

// async = this function is asynchronous (takes time)
async function getData() {
  try {
    // await = wait for the promise to resolve
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    return data;
  } catch (error) {
    // If something goes wrong, catch the error
    console.error('Error:', error);
  }
}

In our project, fetchApi() in api.js uses async/await to call the football API.

Fetch API — Internet Calls

fetch() is a JavaScript function for making HTTP requests (requesting data from a server). It's like typing a URL in the browser, but from code.

// Simple GET request
const response = await fetch('https://api.football-data.org/v4/competitions/WC/matches');
const data = await response.json(); // Convert response to JSON

The project uses fetch in two ways:

  1. Through the PHP proxy (main method) — fetch('proxy.php?endpoint=...')
  2. Direct (fallback) — fetch('https://api.football-data.org/...', { headers: { 'X-Auth-Token': '...' } })

Promise.all — Multiple Calls at Once

When we need to make several API requests that don't depend on each other, we can do them in parallel to save time:

const [data1, data2, data3] = await Promise.all([
  fetch('/api/resource1'),
  fetch('/api/resource2'),
  fetch('/api/resource3')
]);

In knockout.js we use Promise.all to fetch all tournament rounds simultaneously.
